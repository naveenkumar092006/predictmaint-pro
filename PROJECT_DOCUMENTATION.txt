## ML MODELS — DETAILED
================================================================================

1. RANDOM FOREST CLASSIFIER (Failure Prediction)
   Input Features  : Temperature (°C), Vibration (mm/s),
                     Pressure (bar), Operating Hours
   Output          : Failure Probability (0-100%)
   Training Data   : 2000 synthetic samples
   Failure Rules   : Temperature > 90°C OR Vibration > 5.5 mm/s
                     OR Pressure > 7.5 bar OR Hours > 4000
                     OR combinations of above
   Accuracy        : ~100% on synthetic data
   Trees           : 100 estimators, max_depth=8

2. ISOLATION FOREST (Anomaly Detection)
   Input           : Same 4 features
   Output          : Anomaly (YES/NO) + Anomaly Score
   Contamination   : 10% (expects 10% anomalies)
   Purpose         : Detect unusual sensor patterns

3. RANDOM FOREST REGRESSOR (Remaining Useful Life)
   Input           : Same 4 features
   Output          : RUL Days (how many days machine can safely operate)
   Color Coding    : Green > 20 days, Yellow 10-20 days, Red < 10 days
   Training Data   : 2000 synthetic samples with degradation formula

## FAILURE DEDUCTION LOGIC
================================================================================

When failure is predicted, root cause is identified:

Dominant Feature    Failure Type      Root Cause
---------------------------------------------------------
High Temperature  → Overheating     → Temperature exceeded safe threshold
High Vibration    → Bearing Issue   → Vibration exceeded safe threshold
High Pressure     → Valve Blockage  → Pressure exceeded safe threshold
High Hours        → Wear and Tear   → Operating hours exceeded safe limit

Solutions are suggested based on failure type:
  Overheating    : Stop machine, check cooling, inspect lubrication
  Bearing Issue  : Inspect bearings, replace if worn, dynamic balancing
  Valve Blockage : Check relief valves, inspect pipelines, reduce pressure
  Wear and Tear  : Schedule overhaul, replace consumables, lubrication service

## COST ESTIMATION LOGIC
================================================================================

Failure Type      Spare Parts    Labor      Multiplier
--------------------------------------------------------
Overheating       Rs.15,000      Rs.8,000   1 + fail_prob
Bearing Issue     Rs.22,000      Rs.12,000  1 + fail_prob
Valve Blockage    Rs.18,000      Rs.9,000   1 + fail_prob
Wear and Tear     Rs.30,000      Rs.15,000  1 + fail_prob

Preventive Cost   = Total × 0.35
Breakdown Repair  = Total × 1.80
Estimated Savings = Breakdown - Total


## DOWNTIME CALCULATOR LOGIC
================================================================================

Repair Time Ranges:
  Overheating    : 4-12 hours
  Bearing Issue  : 8-24 hours
  Valve Blockage : 3-8 hours
  Wear and Tear  : 24-72 hours

Production Loss = Units Lost × Value Per Unit
Labor Cost      = Hours × Rs.1,500/hour
Penalty Cost    = Revenue Loss × 10%
Total Loss      = Revenue + Labor + Penalty
Savings         = Total Loss × 70% (if preventive)

## OEE CALCULATION
================================================================================

OEE = (Availability × Performance × Quality) / 10000

Availability  = Health Score × 0.95 (±5% random)
Performance   = 100 - Failure_Probability × 0.6 (±3% random)
Quality       = 100 - Failure_Probability × 0.4 (±2% random)

Rating: Excellent ≥ 85%, Good ≥ 70%, Average ≥ 55%, Poor < 55%


## 2FA EMAIL OTP SYSTEM
================================================================================

Flow:
  1. User enters username + password
  2. Credentials verified against SQLite database
  3. 6-digit random OTP generated
  4. OTP stored in memory with 5-minute expiry
  5. OTP sent to user's registered Gmail via SMTP
  6. User redirected to /verify-2fa page
  7. Timer counts down from 5:00
  8. User enters 6-digit OTP
  9. OTP verified (checks used, expired, correct)
  10. Login successful → redirect to dashboard

Email: Professional HTML email with company branding
Fallback: If email not configured → OTP printed to console
