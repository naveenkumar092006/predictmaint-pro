Complete Deployment Guide — From Your Laptop to a Live Website

1
Install Git on your laptop
One-time setup — takes 2 minutes
  1. Go to git-scm.com/downloads → Download for Windows
  2. Run the installer → click Next on everything → Finish
  3. Restart Command Prompt
  4. Verify it worked:
git --version
# Should show: git version 2.43.0 (or similar)
2
Create a GitHub account
Free — needed to host your code
  1. Go to github.com → click "Sign up"
  2. Enter your email, create a password, choose a username
  3. Verify your email (check your inbox)
  4. Choose the FREE plan
3
Extract your project
Get the files ready on your laptop
  1. Find the file: predictive_maintenance_v7_DEPLOY.zip
  2. Right-click → "Extract All" → choose a folder (e.g. C:\Projects\)
  3. You will get a folder called: pm_v7
Inside pm_v7 you should see: app.py, models.py, templates/, static/, etc.
4
Open Command Prompt
The black window where you type commands

Press Win + R on your keyboard → type cmd → press Enter

A black window opens. This is Command Prompt. All commands go here.

5
Navigate to your project folder
Tell Command Prompt where your files are
cd C:\Projects\pm_v7
# Change the path to wherever you extracted the zip

The prompt changes to show C:\Projects\pm_v7>

To verify you are in the right folder, type dir — you should see app.py listed.

6
Set up Git (one time only)
Tell Git who you are
git config --global user.name "Vinay"
git config --global user.email "your@gmail.com"
No output after these commands = success ✅
7
Initialize, add files, and commit
Save your project snapshot in Git
git init
# Creates a Git repository in your folder
git add .
# Adds ALL files (the dot means everything)
git commit -m "PredictMaint Pro v7 Hackathon"
# Saves a snapshot — you'll see a list of files
✅ After commit you'll see: "1 file changed" or many files changed. That means success!
8
Create a repository on GitHub website
This is where your code lives online
  1. Go to github.com → click the + button (top right)
  2. Click "New repository"
  3. Repository name: predictmaint-pro
  4. Select: Public
  5. Do NOT tick "Add README" or any other checkbox
  6. Click the green "Create repository" button
9
Get a GitHub Personal Access Token
GitHub password won't work — you need a token
  1. Click your profile picture (top right on GitHub) → Settings
  2. Scroll all the way down → click "Developer settings"
  3. Personal access tokens → Tokens (classic) → Generate new token (classic)
  4. Note: "Deploy token" | Expiration: 90 days
  5. Tick the box next to repo (top checkbox)
  6. Click "Generate token" at the bottom
  7. COPY the token immediately — starts with ghp_...
⚠️ Save this token somewhere safe! GitHub only shows it ONCE. You will use it as your password.
10
Push your code to GitHub
Upload everything online

GitHub showed you these commands after creating the repo. Replace YOURNAME with your GitHub username:

git remote add origin https://github.com/YOURNAME/predictmaint-pro.git
git branch -M main
git push -u origin main

When it asks:

  1. Username: your GitHub username
  2. Password: paste your Personal Access Token (from Step 9)
✅ Refresh github.com/YOURNAME/predictmaint-pro — you should see all your files!
11
Get Gmail App Password (for OTP)
Your regular Gmail password won't work for sending emails
  1. Go to myaccount.google.com
  2. Click "Security" in the left menu
  3. Under "How you sign in to Google" → click "2-Step Verification" → Enable it
  4. Scroll down → click "App passwords"
  5. Select app: Mail | Select device: Windows Computer
  6. Click "Generate"
  7. Copy the 16-character password (like: abcd efgh ijkl mnop)
💡 Use this 16-char password (without spaces) as MAIL_PASSWORD on Render
12
Get Gemini API Key (for AI ChatBot)
Free — 60 requests per minute
  1. Go to aistudio.google.com
  2. Sign in with your Google account
  3. Click "Get API Key" → "Create API Key"
  4. Copy the key (starts with AIzaSy...)
13

14
Add free PostgreSQL database (optional)
For persistent data storage instead of SQLite
  1. In Render dashboard → "New +" → "PostgreSQL"
  2. Name: predictmaint-db | Plan: Free → "Create Database"
  3. Copy the "Internal Database URL"

Troubleshooting common errors
If something goes wrong — fix it here
  • ❌ "git is not recognized"Git not installed. Restart Command Prompt. Or reinstall from git-scm.com
  • ❌ "Authentication failed" when pushingUse your Personal Access Token (ghp_...) as password — not your GitHub password
  • ❌ "Application error" on RenderClick "Logs" in Render → most common: missing environment variable. Check SECRET_KEY and RENDER=true are set
  • ❌ OTP not arriving in GmailCheck MAIL_PASSWORD is the 16-char App Password (not your Gmail login password). Check spam folder.
  • ❌ App very slow to loadFree Render plan sleeps after 15 min inactivity. First request takes 30-60 sec. Totally normal!
  • ❌ ChatBot says "quota exceeded"Gemini free quota is 60 requests/min. Wait 1 minute. Rule-based fallback still works perfectly.
